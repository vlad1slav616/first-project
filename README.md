
# Тема 1: Введение в Git  

## Что такое Git?  

Git — это распределённая система контроля версий, которая позволяет отслеживать изменения в файлах и координировать работу над проектами между несколькими разработчиками.

## Зачем нужен Git?

* Для отслеживания истории изменений в проекте.
* Для возможности возврата к предыдущим версиям кода.
* Для совместной работы над проектом командой разработчиков.

## Основные понятия:

* Репозиторий (repository) — место, где хранятся все файлы проекта и история их изменений.
* Коммит (commit) — запись об изменениях, сделанных в проекте.
* Ветвь (branch) — независимая линия разработки, которая может быть объединена с другими ветками.
* Индекс (staging area) — промежуточная область, где собираются изменения перед коммитом.

# Тема 2: Работа с Git

## Основные команды Git:

* git init — инициализация нового репозитория Git.
* git status — проверка состояния репозитория (какие файлы изменены, добавлены или удалены).
* git add — добавление файлов в индекс (стадию подготовки к коммиту).
* git commit — сохранение изменений в истории репозитория с комментарием.
* git push — отправка изменений на удалённый сервер.

## Создание репозитория:

* Создание локального репозитория командой git init.
* Добавление файлов в репозиторий и их отслеживание.
* Работа с удалённым репозиторием:
* Связывание локального репозитория с удалённым сервером.
* Отправка изменений на удалённый сервер командой git push.

## Хеш  
Что такое хеш. Хеширование коммитов
Хеширование (от англ. hash, «рубить», «крошить», «мешанина») — это способ преобразовать набор данных и получить их «отпечаток» (англ. fingerprint).
Информация о коммите — это набор данных: когда был сделан коммит, содержимое файлов в репозитории на момент коммита и ссылка на предыдущий, или родительский (англ. parent), коммит.
Git хеширует (преобразует) информацию о коммите с помощью алгоритма SHA-1 (от англ. Secure Hash Algorithm — «безопасный алгоритм хеширования») и получает для каждого коммита свой уникальный хеш — результат хеширования.

## Лог
Лог (Log) - История коммитов в репозитории. - Позволяет отслеживать изменения, просматривать авторов и даты коммитов. - Команда git log отображает историю коммитов с деталями.
Получить сокращённый лог можно с помощью команды git log с флагом --oneline (англ. «одной строкой»). В терминале появятся только первые несколько символов хеша каждого коммита и их комментарии.
Сокращённый лог полезен, если в репозитории уже много коммитов — например, сотни или тысячи. В этом случае можно быстро найти нужный по описанию.
